## 2024-05-24 - Avoid querying large payload from database for lists
**Learning:** Returning all fields from a database query, specifically the `content` field for blog posts in `src/app/api/posts/route.ts`, can severely impact frontend performance due to large payload size. Loading large strings which are not used in listing elements (e.g. infinite scroll) will unnecessarily degrade response time and spike memory usage.
**Action:** When returning list items like in infinite scroll pagination (`/api/posts`), explicitly `select` only the required fields and omit large text fields (like `content`) unless they are strictly required. Use derived fields like `excerpt` or limit content payload if necessary, but omitting the large payload in backend will be much faster. Note that `DynamicPostGrid` attempts to extract an excerpt and image from the `content` via `getExcerpt` and `getFirstImage` within `useMemo`. If `content` is excluded from the API response, these extractions will need to happen on the server, or the API response needs to be adjusted.
