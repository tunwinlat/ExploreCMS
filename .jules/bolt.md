## 2025-02-12 - Prevent unused memoized state and unnecessary loop evaluations
**Learning:** Adding lazy loading with `loading="lazy"` on standard `<img>` tags is a valid optimization. However, when introducing `useMemo` block to compute derived states and format fields (like coverImage or excerpt) prior to returning the map UI output, the actual returned variable in JSX must iterate over this new format. Leaving the original variables mapped over will lead to degraded performance due to unnecessary "expensive regex" evaluations inside `useMemo` on every trigger when that state block is unused.
**Action:** When implementing derived state `useMemo` mappings to prevent regressions during PR evaluations, ensure that the final JSX iteration map is updated to the newly defined variable instance so it does not result in an `unused-vars` linter warning.
