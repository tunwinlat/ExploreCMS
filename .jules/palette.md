## 2025-06-22 - Search Dropdown Keyboard Navigation Event Bubbling
**Learning:** When implementing keyboard navigation (ArrowUp/ArrowDown) on a custom dropdown or modal using an `onKeyDown` handler attached to a container `div` that wraps an `<input>`, the event will bubble up from the input to the container. If the state update function uses a functional updater (e.g., `setSelectedIndex(prev => prev + 1)`), this bubbling can cause the handler to trigger twice for a single key press (once on the input, once on the container), resulting in the selection skipping items.
**Action:** Always either attach the keydown handler directly to the element receiving focus (the input) OR ensure you call `e.stopPropagation()` in the handler if it's placed on a parent container to prevent bubbling loops and double-execution of the logic.
